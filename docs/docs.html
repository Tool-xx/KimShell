// overview

Что такое KimShell

🧬

Поддельный профиль железа

CPU, GPU, RAM, разрешение экрана и User-Agent генерируются случайно каждую сессию. Сайты видят несуществующий компьютер с AMD Ryzen или Intel Core.

🧩

JS-расширение реального времени

Перехватывает Canvas, WebGL, Battery, AudioContext, Plugins, Fonts и десятки других API до того как страница их прочитает. Внедряется на document_start.

🗂️

Изолированный профиль

Каждый запуск — новая временная директория в %APPDATA%\KimShell\. При закрытии все данные перезаписываются в 3 прохода и удаляются.

🛡️

Не трогает основной Brave

Завершение процессов строго по PID (taskkill /T /PID), не по имени. Системный браузер, сессии и закладки пользователя не затрагиваются.

🔐

Secure Drop Zone

Drag & Drop файлов с автоматическим AES-256-GCM шифрованием в карантин. Опасные расширения (.exe, .dll, .js) помечаются автоматически.

🌐

VPN-детект

Проверка публичного IP через несколько серверов, определение VPN/Proxy по ASN и хостингу. Предупреждает о прямом соединении.

// getting started

Установка

  1. Клонировать репозиторий

    bash git clone https://github.com/Tool-xx/KimShell cd kimshell

  2. Установить зависимости

    bash pip install -r requirements.txt

    Требуется Python 3.11+. Зависимости: PyQt6, requests, psutil, pycryptodome.

  3. Запустить

    bash python main.py

    При первом запуске автоматически скачается Brave Browser (~120 МБ) из официального репозитория. При повторных — используется кешированная версия из %APPDATA%\KimShell\brave_portable\.

  4. Опционально: закрыть DNS-утечку

    В Brave: Settings → Privacy and security → Security → Use secure DNS → With: Cloudflare (1.1.1.1)

⚠️ Требования

Windows 10 версии 1903 или новее (64-bit). Windows 7/8 не поддерживаются из-за требований PyQt6 и актуальных версий Brave. Минимум 4GB RAM рекомендуется для стабильной работы.

⚠️ Ограничения

Secure wipe на SSD/NVMe: Из-за wear leveling и контроллера флеш-памяти, перезапись файла физически не гарантирует удаление данных с диска. Для критичных сценариев используй шифрование диска (BitLocker/VeraCrypt).

Canvas fingerprint: Хеш уникален каждую сессию — это нормально. Главное что он не совпадает с твоим реальным браузером и меняется при каждом запуске.

// faq

Вопросы и ответы

  • Brave скачивается каждый раз при запуске?
    Нет. Загрузка происходит только при первом запуске или если brave.exe удалён. Версия кешируется в файле .version внутри директории %APPDATA%\KimShell\brave_portable\. Проверка обновлений: BRAVE_FORCE_UPDATE = False по умолчанию.
  • Закрывает ли KimShell мой основной Brave?
    Нет. Завершение процессов происходит строго по PID (taskkill /F /T /PID <наш_pid>), а не по имени процесса (/IM brave.exe). Системный Brave, твои закладки, история и сессии не затрагиваются. Это критически важный фикс v2.0.
  • Где хранятся данные сессии?
    В %APPDATA%\KimShell\ — поддиректории temp\ (профили), cache\, extension\ (JS-инъекции), brave_portable\ (браузер). Всё удаляется при закрытии если включена опция "Очистка при закрытии". Secure wipe: 3 прохода (0x00 → 0xFF → random).
  • Нужен ли VPN?
    KimShell скрывает аппаратный отпечаток браузера, но не IP-адрес. Для полной анонимности используй VPN вместе с KimShell. DNS-запросы также должны идти через VPN или защищённый DNS (Cloudflare 1.1.1.1).
  • Почему Canvas fingerprint всё равно уникален?
    Canvas noise делает хеш отличным от других пользователей — это и есть цель. Главное что он меняется каждую сессию и не совпадает с твоим реальным браузером. Уникальность в пределах одной сессии — ожидаемое поведение.
  • Работает ли на Windows 7 / 8?
    Нет. Требуется Windows 10 версии 1903 или новее для корректной работы PyQt6 и актуальных версий Brave. Windows 7 не поддерживает необходимые API безопасности.
  • Можно ли использовать свой User-Agent?
    Технически да — модифицируй hardware_spoof.py, метод generate_profile(). Но рекомендуется оставить автогенерацию для реалистичности (Chrome 120-126 на Win10 x64).
  • Что делает Secure Drop Zone?
    Принимает файлы через Drag & Drop или диалог. Опасные расширения (.exe, .dll, .bat, .js, .vbs, .ps1, .msi, .scr) помечаются предупреждением. Файл шифруется AES-256-GCM, сохраняется в карантин (QUARANTINE_DIR), затем расшифровывается для использования. Это изолирует потенциально опасные файлы от основной системы.
